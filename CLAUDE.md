# CLAUDE.md — 全局规则

## 行为规范

1. **Git 操作**：仅在用户主动提及时执行 git 提交、推送等操作，不主动发起。

2. **编码意识**：修改文件前先检查文件中是否含有中文注释。若有中文，编码格式保持原文件编码（常见 GBK/UTF-8），换行符保持原样（CRLF/LF），不转换。

3. **脚本执行环境注意**：当前 Bash 工具运行在 **Git Bash (MINGW64/MSYS2)** 环境下，非原生 Windows cmd。

   **.bat 脚本的中文编码问题**

   - **现象**：在 Git Bash 中通过 `cmd /c` 调用含中文的 `.bat` 脚本时，中文内容解析失败、输出乱码。
   - **根因**：Git Bash 内部使用 UTF-8 处理参数字符串，而 `cmd.exe` 期望系统区域编码（中文 Windows 为 GBK/CP936）。中文内容经 Git Bash 传递给 `cmd` 时编码不匹配，导致字符损坏。
   - **避免方法**：

   | 场景 | 做法 |
   |------|------|
   | 新建脚本 | **首选 Python 脚本**（`.py`），跨平台、原生 Unicode，不存在编码问题。直接 `python script.py` 运行 |
   | 无法用 Python | 使用 `.sh`（POSIX shell），Git Bash 原生执行 |
   | 必须使用 `.bat` | 脚本内避免中文；或用英文替代；或确保文件编码为 GBK |

   > **原则：脚本优先选 `.py` → `.sh` → `.bat`（按编码安全性排序）。**

4. **禁止使用 Worktree**：所有操作直接在项目主目录进行，不创建或使用 git worktree。

5. **修改原则**：
   - 先理解现有代码结构和风格，再动手修改
   - 遵循项目现有的设计模式，不引入不必要的抽象
   - 不做格式化或重构性质的额外改动
   - 修改前先读取要编辑的文件，确认格式

6. **中间文件管理**：执行任务过程中产生的临时脚本、中间文件等，需创建单独的 `_tmp/` 子目录来存放，不要直接放到工作目录下。

7. **跨项目工作**：在多个项目目录间切换工作时，注意每个项目可能有自己的 CLAUDE.md 项目规则，项目级规则优先于本全局规则。
